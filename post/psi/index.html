基于RSA的PSI协议

2021-10-14
加密算法

协议流程图

FATE中的隐私求交（PSI）基于RSA技术，协议流程如下图所示。

Client方持有的ID集合记为$X_A$：{$u_1,u_2,u_3,u_4$}，Host方持有的ID集合记为$X_B$：{$u_0,u_2,u_3,u_5$}。

协议目标：Client和Host双方都获得交集{$u_2,u_3$}，Client和Host均无法获得对方交集外ID，即Client无法获得{$u_0,u_5$}，Host无法获得{$u_1,u_4$}。

协议过程

  1. Host方生成RSA密钥(e,d,n)。

  2. Host将公钥(e,n)发送给Client。

  3. Client基于收到的公钥，针对每个ID $u_i$，生产一个随机数$r_i$，然后对原始ID$u_i$进行“盲化”:$r_i^e\times H(u_i)\mod n$ 。其中，$H(u_i)$表示对原始id$u_i$进行hash运算。盲化结果记为$X_A'$。

$X_A'$ = {$(r_1^e\times H(u_1))\% n, (r_2^e\times H(u_2))\% n, (r_3^e\times H(u_3))\% n, (r_4^e\times H(u_4))\% n$}

注：图中式中去除了“$\mod n(\% n)$”，仅为简化起见，实际计算不可缺少

  1. Client将$X_A'$发给Host。

  2. Host收到$X_A'$后，做两件事情：

    a. 利用私钥d，对$X_A'$内元素逐一加密（或曰签名），$(r_i^e\times H(u_i))^d\% n = (r_i\times (H(u_i))^d)\% n$，结果集记为$Z_A$。

    $Z_A$ = {$(r_1\times (H(u_1))^d)\%n, (r_2\times (H(u_2))^d)\%n, (r_3\times (H(u_3))^d)\%n, (r_4\times (H(u_4))^d)\%n$}

    b. 利用私钥d，对己方的每个$H(u_j)$进行签名，即$(H(u_j))^d\mod n$，然后，针对此结果进行二次Hash，得$H((H(u_j))^d\mod n)$，结果集记为$Z_B$。

    $Z_B$ = {$H(H(u_0))^d)\%n), H(H(u_2))^d)\%n), H(H(u_3))^d)\%n), H(H(u_5))^d)\%n)$}

  3. Host将$Z_A$和$Z_B$发送给Client。

  4. Client对$Z_A$中元素进行“除盲”操作：首先乘以$r_i$的逆元，再对结果进行Hash：$H(r_i\times r_i^{-1}\times(H(u_i))^d \mod n)$=$H((H(u_i))^d \mod n)$。结果集记为$D_A$。

$D_A$ = {$H(H(u_1))^d)\%n), H(H(u_2))^d)\%n), H(H(u_3))^d)\%n), H(H(u_4))^d)\%n)$}

  1. Client计算$D_A$和$Z_B$的交集:$D_A \bigcap D_B = {H(H(u_2))^d)\%n), H(H(u_3))^d)\%n)}$。因此，Client能知道原始交集为{$u_2, u_3$}。

  2. Client将交集结果{$u_2, u_3$}同步给Host。

问题1:交集外ID如何被保护？

求交运算基于$D_A$和$Z_B$，而$Z_B$中的数据经Host私钥加密，因此Guest无法从$Z_B$中得到Host的原始ID值。另外，$X_A'$中的元素，都经随机数$r_i$盲化处理，因此Host无法从$X_A'$中得到Client的原始ID。

最终$D_A$和$Z_B$的交集，Client可根据交集元素的index确定原始交集ID，但$Z_B$中交集外ID，Client无法获知。

问题2:步骤5.b中二次Hash的作用？

如果不进行二次Hash，Host直接将$(H(u_j))^d\mod n$发给Client，协议仍可走通（只要在第7步中，Client也不进行二次Hash），但是问题在于：Client可以利用公钥e进行解密，得到$H(u_j)$，从而可以碰撞出$u_j$，因而安全性无法满足。
