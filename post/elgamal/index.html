Elgamal加密算法

2021-09-18
加密算法

从素数说起

素数又称为质数，是指大于1的整数，且除了1和它本身以外，不能被其他任何整数整除。在英文中，素数被称为“prime number”，“prime”这个单词的意思是“基础的，基本的”，因此，“prime number”表达的实际含义是“基础数”。之所以如此得名，则来源于著名的“算术基本定理”。

算术基本定理：每个大于1的自然数，要么本身就是素数，要么可以写为2个或以上的素数的积。
例如：$10 = 2 \times 5$，$1200 = 2^4 \times 3 \times 5^2 $。

可见，任意的正整数，都可以通过因式分解的方式，不断拆分成更小的整数，直到无法进一步拆分。换句话说，对于任意正整数，都可以用若干“无法再拆分”的数相乘来构造，这些“无法再拆分”的整数，可以视作所有整数的基础，因此得名“prime number”，中文翻译成素数或质数，其实不太直接，“素”和“质”可以意会为“基础”，“本质”。

“prime”还有一层含义是“优质的，一流的”，而从密码学的角度看，“prime number”确实非常“prime”，“质数”确实非常“优质”。

对于任意素数$p$，都能找到一个整数$g$，使得对于不同的整数$i$，通过计算$g^i\ mod(p)$，能够得到从$1$到$p-1$范围内所有整数。
例如：$p=5$，$g=2$，$2^1 mod(5) = 2,2^2 mod(5) = 4,2^3 mod(5) = 3,2^4 mod(5) = 1$。

事实上，这种通过$g^i\ mod(p)$计算得到的整数集合，称为循环群，$g$称为该群的生成元，集合的元素个数称为群的阶数。当$p$是素数时，一定可以选取到某个生成元，使得循环群的阶数达到$p-1$。如上例，$p=5$时，选取$g=2$作为生成元，得到的循环群的阶数为4（即$p-1$）。

而对于非素数，就没有这么好的“运气”了：利用同样方式构造出的循环群，其阶数往往很小。例如取$p=6$，$g=2$，可以发现：$2^1 mod(6) = 2,2^2 mod(6) = 4,2^3 mod(6) = 2$，得到的循环群为{$2, 4$}，阶数为2，小于5。如果取$g=3$，则有：$3^1 mod(6) = 3,3^2 mod(6) = 3$，得到的循环群为{3}，阶数小到只有1。

素数的这种特性，使其成为很多加密算法的基础。

Diffie-Hellman密钥交换

Diffie-Hellman密钥交换（简称DH密钥交换）协议，可以让协议双方在不安全的信道上，以安全的方式确立一个双方共享的密钥。

假设双方为Alice和Bob，协议流程如下：

  1. Alice和Bob协商一个素数$p$和生成元$g$。
  2. Alice随机选择一个自然数$a$，将$g^a\ mod(p)$发送给Bob，但将$a$本身保密。
  3. Bob随机选择一个自然数$b$，将$g^b\ mod(p)$发送给Alice，但将$b$本身保密。
  4. Alice计算$(g^b)^a\ mod(p)$。
  5. Bob计算$(g^a)^b\ mod(p)$。

由于$(g^b)^a=(g^a)^b=g^{ab}$，因此Alice和Bob以$g^{ab}\ mod(p)$作为共享密钥。

DH密钥交换协议的安全性来自于几个方面，首先是所谓的离散对数难题，即：在已知$g, p, g^a\ mod(p)$三个数值的情况下，求出$a$是非常困难的事情（当然，前提是$p$是一个很大的数，无法暴力求解）。因此以上协议中，$g, p, g^a\ mod(p), g^b\ mod(p)$可以通过不安全信道发送，而不用担心$a$和$b$被反算出来。

DH协议安全性还来自于$p$的取值。首先，$p$一定会是一个很大的数，否则攻击者可以通过暴力求解的方式，从$g^a\ mod(p)$中反算出$a$；此外，$p$一定会是一个素数，因为基于素数构成的循环群阶数可达$p-1$，这增加了密钥$g^{ab}\ mod(p)$被猜中的难度。

Elgamal加密

Elgamal加密是基于DH密钥交换的非对称加密算法。依然假设双方为Alice和Bob，加密与解密流程如下：

  1. Alice和Bob协商一个素数$p$和生成元$g$。
  2. Alice随机选择一个自然数$a$，将$g^a\ mod(p)$发送给Bob，但将$a$本身保密。
  3. Bob加密：Bob随机选择一个自然数$b$，计算$(g^a)^b$，将$g^b\ mod(p)$以及$m(g^a)^b\ mod(p)$发送给Alice，但将$b$本身保密。
  4. Alice解密：Alice先计算$(g^b)^a\mod(p)$,然后再计算其逆元，从而得到$\frac{m(g^a)^b\mod(p)}{(g^b)^a\mod(p)} = m$。

Elgamal加密具有乘法同态特性。假设消息$m_1$和$m_2$,对应密文$c_1$和$c_2$分别为$m_1(g^a)^b\ mod(p)$，$m_2(g^c)^d\ mod(p)$（假设两次随机数不同，分别为$a,b$和$c,d$），则：

$c_1*c_2 = m_1m_2(g^a)^b(g^c)^d$

因此：

$D(c_1*c_2)=D(c_1)*D(c_2)$

即，两个数加密之后相乘，结果等于两个数相乘之后再加密。

由于$a$和$b$可以每次加密随机选取，因此对于相同的消息$m$，每次加密结果都不同。

模运算$\mod(p)$

在DH密钥交换中，$(g^b)^a=(g^a)^b=g^{ab}$，为何Alice和Bob不直接使用$g^{ab}$，而是使用模运算的结果$g^{ab}\ mod(p)$作为共享密钥？ 因为$a$和$b$的取值往往很大，因此$g^{ab}$的值会异常大，大到无法存储和计算。

而因为$mod(p)$的引入，从而产生了对明文$m$的长度要求：其位数不能超过$p$的位数。
